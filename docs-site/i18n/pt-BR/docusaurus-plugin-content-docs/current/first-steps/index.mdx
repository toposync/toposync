# Primeiros passos {/* #first-steps */}

Esta seção é um caminho prático para usar o Toposync pela primeira vez. Ela é intencionalmente simples: criar uma composição, adicionar uma base visual, desenhar a estrutura principal, adicionar uma câmera e depois criar o primeiro pipeline básico.

O Toposync ainda é um software alpha. Use redes contidas, dispositivos de teste, entidades não críticas do Home Assistant e as orientações de segurança em [Política de segurança](https://github.com/toposync/toposync/blob/main/SECURITY.md) ao avaliar o produto.

## Ordem recomendada {/* #recommended-order */}

1. [Crie sua primeira composição](create-your-first-composition.mdx).
2. Adicione uma imagem de decalque, como uma planta baixa ou mapa de robô aspirador.
3. [Adicione paredes e áreas](add-walls-and-areas.mdx).
4. [Adicione um modelo 3D leve](add-a-3d-model.mdx), se isso ajudar na visualização espacial.
5. [Conecte o Home Assistant](connect-home-assistant.mdx), se quiser entidades e estados na composição.
6. [Adicione sua primeira câmera](add-your-first-camera.mdx).
7. [Crie seus primeiros pipelines](first-pipelines.mdx).
8. Revise [próximos passos](next-steps.mdx) quando o primeiro ciclo funcionar.

## Comece pequeno {/* #start-small */}

Não comece importando tudo que você tem. Um bom primeiro teste é:

- uma composição;
- um andar ou área da casa;
- uma imagem de decalque;
- algumas paredes ou áreas;
- uma câmera;
- um pipeline simples.

Depois que isso funcionar, adicione mais cômodos, câmeras, entidades e fluxos aos poucos.

## O que importa primeiro {/* #what-matters-first */}

O primeiro objetivo não é perfeição visual. O primeiro objetivo é significado espacial:

- onde cada área fica;
- onde a câmera está;
- o que a câmera consegue ver;
- quais eventos pertencem a cada área;
- quais entidades do Home Assistant são úteis naquele contexto.

Quando isso fica claro, a interface se torna mais fácil de melhorar.
