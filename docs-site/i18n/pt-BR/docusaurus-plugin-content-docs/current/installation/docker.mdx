# Docker {/* #docker */}

Use Docker quando quiser rodar o Toposync isolado do sistema host e com operação mais fácil de reconstruir ou mover entre máquinas.

O Toposync publica imagens oficiais no GitHub Container Registry:

```text
ghcr.io/toposync/toposync
```

Imagens públicas no GHCR podem ser baixadas sem login no Docker ou no GitHub.

## Escolha Um Caminho Docker

| Cenário | Guia recomendado |
| --- | --- |
| Servidor comum, sem GPU | [Docker CPU](docker-cpu.mdx) |
| Host Linux com GPU NVIDIA | [Docker CUDA](docker-cuda.mdx) |
| Processing server remoto em container | [Processing server em Docker](processing-server-docker.mdx) |

## Recomendação Padrão

Comece por [Docker CPU](docker-cpu.mdx).

Vá para [Docker CUDA](docker-cuda.mdx) somente quando tiver um host Linux com GPU NVIDIA e uma carga real de vision que se beneficie de aceleração.

## O Que Docker Muda

- O Toposync roda em um container.
- Os dados de runtime são montados do host para `/data`.
- A UI e a API continuam sendo servidas pela mesma porta pública.
- Suporte a GPU fica em um caminho CUDA separado, não no container CPU padrão.
- Suporte a streaming já vem incluído nas imagens públicas CPU e CUDA.

Para suporte por sistema e arquitetura, consulte [Compatibilidade](architecture-support.mdx).
