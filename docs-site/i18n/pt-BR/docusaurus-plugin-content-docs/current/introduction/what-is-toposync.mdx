# O que é o Toposync {/* #what-is-toposync */}

Toposync é uma plataforma open source, local-first, para **Spatial Home Automation com inteligência local**. Ele reúne câmeras, áreas, dispositivos, entidades do Home Assistant, composições 2D e 3D e pipelines em um modelo espacial para ambientes residenciais privados.

O projeto parte de uma ideia simples: automação residencial fica mais útil quando eventos têm lugar, câmeras têm contexto e automações podem reagir à área onde algo aconteceu.

## Qual problema ele resolve {/* #what-problem-it-solves */}

A maior parte dos sistemas de automação residencial e câmera é organizada como listas:

- listas de dispositivos;
- listas de entidades;
- feeds de câmera;
- regras de automação;
- históricos de eventos;
- fluxos de notificações.

Essas listas são úteis, mas muitas vezes perdem o contexto físico. O Toposync adiciona uma camada mapeada onde câmera, área, evento, dispositivo e automação podem ser entendidos juntos.

Isso ajuda a responder perguntas como:

- onde aconteceu?
- qual câmera viu?
- qual área foi afetada?
- qual dispositivo próximo deveria reagir?
- isso deve virar notificação, imagem armazenada, item de histórico ou saída para Home Assistant?

## O que o Toposync não é {/* #what-toposync-is-not */}

Toposync não substitui Home Assistant, ferramentas RTSP, utilitários ONVIF, dashboards de câmera ou sistemas de segurança certificados.

Ele também ainda não está pronto para ser a única camada de segurança crítica, controle de acesso ou fluxos de emergência. No alpha atual, trate o Toposync como uma camada adicional de inteligência local e visualização que precisa ser testada com cuidado.

## O que ele inclui hoje {/* #what-it-includes-today */}

O alpha atual foca em:

- criar uma composição 2D/3D da casa;
- adicionar paredes, áreas, imagens, modelos, câmeras e contexto do Home Assistant;
- mapear visões de câmera na composição;
- criar pipelines de câmera e visão;
- produzir eventos espaciais, notificações, imagens armazenadas e saídas para Home Assistant;
- delegar cargas pesadas de câmera e visão para processing servers quando necessário.

O caminho padrão do produto é local-first. O servidor principal, a UI, extensões, configuração, arquivos e estado de pipelines rodam no próprio ambiente do usuário.

## Quando faz sentido {/* #when-it-makes-sense */}

Toposync faz mais sentido quando:

- você já usa Home Assistant e quer contexto visual mais rico;
- você tem câmeras IP, RTSP, ONVIF ou dispositivos PTZ;
- você quer eventos de câmera associados a áreas da casa;
- você quer visão computacional local sem depender de um único fabricante de câmera;
- você quer um lugar central para experimentar pipelines de câmera, automação e espaço;
- você está confortável testando software alpha em um ambiente contido.

Se você só precisa de um visualizador básico de câmera ou um dashboard simples de dispositivos, o Toposync pode ser mais do que você precisa hoje.
