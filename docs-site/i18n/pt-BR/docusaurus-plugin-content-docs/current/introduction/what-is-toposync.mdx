# O que é o Toposync {/* #what-is-toposync */}

Toposync é uma plataforma open source, local-first, para **Spatial Home Automation com inteligência local**. Ele reúne câmeras, áreas, dispositivos, entidades do Home Assistant, composições 2D/3D e pipelines em um modelo espacial para ambientes residenciais privados.

O objetivo não é substituir Home Assistant, ferramentas RTSP, utilitários ONVIF ou dashboards de câmera. O objetivo é adicionar uma camada espacial onde esses sistemas consigam responder perguntas mais úteis:

- onde um evento aconteceu?
- qual câmera viu?
- qual área mapeada foi afetada?
- qual objeto, dispositivo ou automação está relacionado?
- o que deve ser armazenado, exibido ou encaminhado?

## A categoria {/* #the-category */}

**Spatial Home Automation** é a categoria de produto que o Toposync está construindo. Ela trata a casa como um ambiente mapeado, não apenas como uma lista de dispositivos ou feeds de câmera.

Dentro dessa categoria, o Toposync usa alguns termos específicos:

- **Spatial Camera Mapping**: conectar a imagem de uma câmera a posições, áreas e visões reais dentro da casa mapeada.
- **Spatial Events**: eventos enriquecidos com posição, área, câmera, estado do objeto, lifecycle, contexto de imagem e contexto de automação relacionado.
- **Spatial Intelligence**: a camada local mais ampla de interpretação que pode combinar entrada de câmera, áreas, dispositivos, pipelines, histórico e futuros serviços opcionais de cloud.
- **Spatial Awareness**: linguagem de recurso para comportamento sensível a áreas, como fazer a mesma detecção significar algo diferente perto de um portão, calçada, piscina, quintal, garagem ou entrada.

Esses termos são linguagem de produto. Labels operacionais continuam precisos: RTSP, ONVIF, PTZ, pipelines, processing servers, áreas e mapeamento de câmera ainda significam os recursos técnicos concretos que descrevem.

## O que o Toposync inclui hoje {/* #what-toposync-includes-today */}

O alpha atual foca em:

- construir uma composição 2D/3D da casa;
- adicionar paredes, áreas, imagens, modelos, câmeras e contexto do Home Assistant;
- mapear câmeras na composição;
- criar pipelines de câmera e visão;
- produzir eventos, notificações, imagens e saídas para Home Assistant;
- delegar processamento pesado para processing servers remotos quando necessário.

O Toposync é local-first por padrão. Dados do usuário, configuração de câmeras, arquivos gerados e artifacts de pipeline ficam no diretório de dados configurado, salvo quando o usuário escolhe explicitamente outro caminho de integração.
